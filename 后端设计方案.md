## 二、数据库设计

### 2.1 表结构设计

#### 2.1.1 users (用户表)
```sql
CREATE TABLE users (
    id VARCHAR(50) PRIMARY KEY,
    account VARCHAR(100) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    name VARCHAR(100) NOT NULL,
    role VARCHAR(50) NOT NULL,
    department VARCHAR(100),
    avatar TEXT,
    source VARCHAR(20) DEFAULT 'LOCAL',
    sso_connected BOOLEAN DEFAULT FALSE,
    is_primary_approver BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 2.1.2 okrs (OKR表)
```sql
CREATE TABLE okrs (
    id VARCHAR(50) PRIMARY KEY,
    user_id VARCHAR(50) NOT NULL REFERENCES users(id),
    user_name VARCHAR(100) NOT NULL,
    level VARCHAR(20) NOT NULL, -- COMPANY, DEPARTMENT, PERSONAL
    department VARCHAR(100),
    title VARCHAR(500) NOT NULL,
    period VARCHAR(50),
    status VARCHAR(50) NOT NULL,
    parent_okr_id VARCHAR(50) REFERENCES okrs(id),
    peer_reviewers TEXT[], -- 存储用户ID数组
    total_score DECIMAL(5,2),
    final_grade VARCHAR(10), -- S, A, B, C
    adjustment_reason TEXT,
    is_performance_archived BOOLEAN DEFAULT FALSE,
    objectives JSONB NOT NULL, -- 存储目标数组
    overall_self_assessment JSONB, -- {score: number, comment: string}
    overall_manager_assessment JSONB, -- {score: number, comment: string}
    cc_feedback JSONB[], -- 存储反馈数组
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 2.1.3 workflows (审批流程表)

```sql
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
```


#### 2.1.5 departments (部门表)
```sql
CREATE TABLE departments (
    id SERIAL PRIMARY KEY,

    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

```

#### 2.1.6 custom_roles (自定义角色表)
```sql
CREATE TABLE custom_roles (
```

## 五、核心功能实现要点

### 5.1 认证与授权
- 使用 JWT 进行用户认证
- 密码使用 bcryptjs 加密存储
- 所有 API（除登录外）都需要 JWT 认证
- 管理员可以预览其他用户视角（前端实现，后端只提供数据）

### 5.2 数据同步
- 前端使用 `alignflow_data_updated` 事件进行数据同步
- 后端所有写操作成功后，前端会触发该事件
- 前端使用同步版本函数（如 `getOKRsSync()`）从缓存读取数据

### 5.3 OKR 数据结构
- `objectives` 使用 JSONB 存储，支持复杂的嵌套结构
- `cc_feedback` 使用 JSONB 数组存储
- `peer_reviewers` 使用 PostgreSQL 数组类型存储

### 5.4 审批流程
- 基于角色的动态审批流程
- 支持 L1、L2、L3 三级审批
- 支持 CC（抄送）角色
- 支持自定义角色

### 5.5 绩效评估
- 支持自评和上级评分
- 支持多级审批
- 支持绩效归档
- 支持等级配置（S/A/B/C）

## 八、注意事项

1. **数据一致性**: 确保前端和后端数据结构完全一致
2. **错误处理**: 所有 API 返回统一的错误格式
3. **安全性**: 
   - 密码加密存储
   - JWT token 过期时间设置
   - SQL 注入防护（使用参数化查询）
4. **性能**: 
   - 合理使用数据库索引
   - 避免 N+1 查询问题
5. **兼容性**: 确保与前端现有代码 100% 兼容
