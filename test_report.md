## 5. 发现与修复的问题

在测试执行过程中，我们识别并修复了以下问题：

1.  **异步错误处理缺失**:
    *   *问题*: 后端 Controller 中未捕获 `async` 函数抛出的异常，导致测试超时挂起。
    *   *修复*: 引入 `express-async-errors` 库，确保所有异步异常被全局错误处理器捕获并返回正确的 HTTP 状态码。

2.  **测试数据一致性**:
    *   *问题*: 部分测试用例使用了系统预设用户（Admin），但在某些特定流程（如部门匹配）中导致校验失败。
    *   *修复*: 调整测试用例，使用具有真实部门层级关系的测试账号（如 `wangxiaogong` -> `liyanfa`）进行测试，确保符合业务逻辑。

3.  **否决逻辑验证**:
    *   *问题*: 初始测试脚本对“否决”后的目标状态假设不明确。
    *   *确认*: 验证了后端逻辑：L2 否决默认回退给上一级（L1/Manager），即 `PENDING_MANAGER` 状态。
