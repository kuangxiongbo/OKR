# 第三方集成配置说明

## 概述

系统已集成企业微信和 SSO 单点登录功能，支持多种第三方认证方式。

## 功能特性

### 1. 企业微信登录
- 支持企业微信 OAuth2.0 授权登录
- 自动创建或更新用户信息
- 支持用户头像和基本信息同步

### 2. SSO 单点登录
- 支持 OAuth2.0 协议
- 支持 CAS 协议
- 支持 SAML 2.0 协议（需要额外库支持）

## 配置步骤

### 1. 数据库迁移

已自动添加以下字段到 `users` 表：
- `wechat_userid`: 企业微信用户 ID
- `wechat_openid`: 企业微信 OpenID
- `sso_provider`: SSO 提供商（WECHAT, SAML, OAUTH2, CAS）
- `sso_id`: SSO 系统用户 ID
- `sso_attributes`: SSO 系统返回的用户属性（JSONB）

运行迁移：
```bash
cd backend
npm run migrate
```

### 2. 环境变量配置

在 `backend/.env` 文件中添加以下配置：

#### 企业微信配置
```env
# 企业微信配置
WECHAT_CORP_ID=your_corp_id                    # 企业 ID
WECHAT_CORP_SECRET=your_corp_secret            # 应用密钥
WECHAT_AGENT_ID=your_agent_id                  # 应用 ID
WECHAT_REDIRECT_URI=http://localhost:3000/auth/wechat/callback  # 回调地址
```

#### SSO 配置
```env
# SSO 基础配置
SSO_ENABLED=true                                # 是否启用 SSO
SSO_PROVIDER=OAUTH2                             # SSO 提供商：OAUTH2, CAS, SAML
SSO_CALLBACK_URL=http://localhost:3001/api/v1/auth/sso/callback  # 回调地址

# OAuth2 配置（当 SSO_PROVIDER=OAUTH2 时）
SSO_OAUTH2_CLIENT_ID=your_client_id
SSO_OAUTH2_CLIENT_SECRET=your_client_secret
SSO_OAUTH2_AUTH_URL=https://sso.example.com/oauth2/authorize
SSO_OAUTH2_TOKEN_URL=https://sso.example.com/oauth2/token
SSO_OAUTH2_USERINFO_URL=https://sso.example.com/oauth2/userinfo

# CAS 配置（当 SSO_PROVIDER=CAS 时）
SSO_CAS_SERVER_URL=https://cas.example.com/cas

# SAML 配置（当 SSO_PROVIDER=SAML 时，需要额外库支持）
SSO_ISSUER=https://sso.example.com
SSO_ENTRY_POINT=https://sso.example.com/sso/login
SSO_CERT_PATH=/path/to/cert.pem
```

#### 前端 URL 配置
```env
FRONTEND_URL=http://localhost:3000              # 前端地址，用于回调重定向
```

### 3. 企业微信应用配置

1. 登录企业微信管理后台
2. 创建应用，获取 `corp_id`、`corp_secret`、`agent_id`
3. 配置应用的回调域名（如：`localhost:3000`）
4. 设置应用可见范围（选择需要登录的用户）

### 4. SSO 系统配置

#### OAuth2 配置
1. 在 SSO 系统中创建 OAuth2 应用
2. 配置回调地址：`http://your-domain:3001/api/v1/auth/sso/callback`
3. 获取 `client_id` 和 `client_secret`
4. 配置授权范围：`openid profile email`

#### CAS 配置
1. 配置 CAS 服务器地址
2. 配置回调地址：`http://your-domain:3001/api/v1/auth/sso/callback`

## 使用说明

### 前端登录页面

登录页面已自动显示第三方登录按钮：
- **企业微信登录**：点击后跳转到企业微信授权页面
- **SSO 登录**：点击后跳转到 SSO 登录页面

### 用户创建规则

- **首次登录**：系统自动创建新用户，默认角色为 `EMPLOYEE`
- **再次登录**：系统自动更新用户信息（姓名、头像等）
- **账号绑定**：管理员可以在后台将第三方账号绑定到现有账号

### 权限管理

- 第三方登录的用户默认角色为 `EMPLOYEE`
- 管理员可以在"用户管理"页面修改用户角色和权限
- 支持将第三方账号与现有本地账号绑定

## API 端点

### 企业微信
- `GET /api/v1/auth/wechat/authorize` - 获取授权 URL
- `GET /api/v1/auth/wechat/callback` - 处理回调

### SSO
- `GET /api/v1/auth/sso/authorize` - 获取授权 URL（或直接重定向）
- `GET /api/v1/auth/sso/callback` - 处理 OAuth2/CAS 回调
- `POST /api/v1/auth/sso/callback` - 处理 SAML 回调

## 安全注意事项

1. **State 参数**：OAuth 流程中使用 state 参数防止 CSRF 攻击
2. **Token 验证**：所有第三方返回的 token/assertion 都会进行验证
3. **HTTPS**：生产环境必须使用 HTTPS
4. **密钥管理**：敏感信息（如 `corp_secret`、`client_secret`）应使用环境变量，不要提交到代码仓库

## 故障排查

### 企业微信登录失败
1. 检查 `WECHAT_CORP_ID`、`WECHAT_CORP_SECRET`、`WECHAT_AGENT_ID` 是否正确
2. 检查回调地址是否在企业微信后台配置
3. 检查应用可见范围是否包含当前用户

### SSO 登录失败
1. 检查 `SSO_ENABLED` 是否为 `true`
2. 检查 `SSO_PROVIDER` 是否与 SSO 系统匹配
3. 检查回调地址是否在 SSO 系统中配置
4. 检查 OAuth2/CAS 配置是否完整

### 用户创建失败
1. 检查数据库连接是否正常
2. 检查用户表是否有足够的权限
3. 查看后端日志获取详细错误信息

## 测试

### 测试企业微信登录
1. 配置企业微信环境变量
2. 启动后端服务：`cd backend && npm run dev`
3. 启动前端服务：`npm run dev`
4. 访问登录页面，点击"企业微信登录"
5. 在企业微信中授权
6. 验证是否成功登录并跳转到首页

### 测试 SSO 登录
1. 配置 SSO 环境变量
2. 启动服务
3. 访问登录页面，点击"SSO 登录"
4. 在 SSO 系统中登录
5. 验证是否成功登录并跳转到首页

## 后续扩展

如需支持更多 SSO 协议或第三方登录：
1. 在 `backend/src/services/ssoAuth.ts` 中添加新的协议支持
2. 在 `backend/src/controllers/authController.ts` 中添加对应的处理逻辑
3. 更新前端登录页面，添加新的登录按钮
