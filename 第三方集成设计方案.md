# 第三方集成设计方案

## 1. 架构设计

### 1.1 集成目标
- **企业微信集成**：支持企业微信 OAuth2.0 授权登录
- **SSO 单点登录**：支持 SAML 2.0、OAuth2.0、CAS 等协议

### 1.2 技术架构

```
前端 (React)
  ↓
登录页面 → 第三方登录按钮 → 跳转到第三方授权页面
  ↓
回调处理 → 后端验证 → 生成 JWT Token → 登录成功
```

### 1.3 数据模型扩展

#### 用户表扩展字段
```sql
ALTER TABLE users ADD COLUMN IF NOT EXISTS wechat_userid VARCHAR(255);
ALTER TABLE users ADD COLUMN IF NOT EXISTS wechat_openid VARCHAR(255);
ALTER TABLE users ADD COLUMN IF NOT EXISTS sso_provider VARCHAR(50); -- 'WECHAT', 'SAML', 'OAUTH2', 'CAS'
ALTER TABLE users ADD COLUMN IF NOT EXISTS sso_id VARCHAR(255); -- 第三方系统用户ID
ALTER TABLE users ADD COLUMN IF NOT EXISTS sso_attributes JSONB; -- 第三方系统返回的用户属性
```

### 1.4 认证流程

#### 企业微信登录流程
1. 用户点击"企业微信登录"
2. 前端跳转到企业微信授权页面
3. 用户在企业微信中授权
4. 企业微信回调到后端 `/v1/auth/wechat/callback`
5. 后端获取用户信息，查找或创建用户
6. 生成 JWT Token 返回
7. 前端保存 Token，完成登录

#### SSO 登录流程
1. 用户点击"SSO 登录"
2. 前端跳转到 SSO 登录页面（或直接重定向）
3. SSO 系统验证用户身份
4. SSO 回调到后端 `/v1/auth/sso/callback`
5. 后端验证 SAML Assertion / OAuth Token
6. 查找或创建用户
7. 生成 JWT Token 返回
8. 前端保存 Token，完成登录

## 2. 后端实现

### 2.1 企业微信集成

#### 依赖包
```json
{
  "axios": "^1.6.0",
  "jsonwebtoken": "^9.0.0"
}
```

#### 配置项（环境变量）
```env
# 企业微信配置
WECHAT_CORP_ID=your_corp_id
WECHAT_CORP_SECRET=your_corp_secret
WECHAT_AGENT_ID=your_agent_id
WECHAT_REDIRECT_URI=http://localhost:3000/auth/wechat/callback

# SSO 配置
SSO_ENABLED=true
SSO_PROVIDER=SAML # SAML, OAUTH2, CAS
SSO_ISSUER=https://sso.example.com
SSO_CERT_PATH=/path/to/cert.pem
SSO_ENTRY_POINT=https://sso.example.com/sso/login
SSO_CALLBACK_URL=http://localhost:3001/api/v1/auth/sso/callback
```

### 2.2 API 端点设计

```
GET  /v1/auth/wechat/authorize     # 获取企业微信授权 URL
GET  /v1/auth/wechat/callback       # 企业微信回调处理
GET  /v1/auth/sso/authorize         # 获取 SSO 授权 URL
POST /v1/auth/sso/callback           # SSO 回调处理（SAML POST）
GET  /v1/auth/sso/callback           # SSO 回调处理（OAuth2 GET）
```

## 3. 前端实现

### 3.1 登录页面改造
- 添加"企业微信登录"按钮
- 添加"SSO 登录"按钮
- 保留原有账号密码登录

### 3.2 回调页面
- `/auth/wechat/callback` - 企业微信回调处理
- `/auth/sso/callback` - SSO 回调处理

## 4. 安全考虑

1. **State 参数**：OAuth 流程中使用 state 参数防止 CSRF 攻击
2. **Token 验证**：验证第三方返回的 token/assertion
3. **用户绑定**：首次登录时，需要绑定现有账号或创建新账号
4. **权限控制**：第三方登录用户需要分配相应的角色和权限
